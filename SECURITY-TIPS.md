# SECURITY TIPS

## CIA Triad
- **Confidentiality**: Ensure that sensitive information is accessible only to those authorized to have access. (Polish: **Poufność**: Upewnij się, że wrażliwe informacje są dostępne tylko dla osób upoważnionych do ich posiadania.)
- **Integrity**: Safeguard the accuracy and completeness of information. (Polish: **Integralność**: Chroń dokładność i kompletność informacji.)
- **Availability**: Ensure that authorized users have access to information and associated assets when required. (Polish: **Dostępność**: Upewnij się, że upoważnieni użytkownicy mają dostęp do informacji i powiązanych zasobów, gdy ich potrzebują.)

## Principle of Least Privilege
- Grant users the minimum level of access necessary to perform their job functions. (Polish: **Zasada minimalnych uprawnień**: Przyznawaj użytkownikom minimalny poziom dostępu niezbędny do wykonywania ich funkcji zawodowych.)

## Defense in Depth
- Employ multiple layers of security controls. (Polish: **Ochrona warstwowa**: Zastosuj wiele warstw kontroli bezpieczeństwa.)

## IP Security
- Use IPsec to secure IP communications. (Polish: **Bezpieczeństwo IP**: Używaj IPsec do zabezpieczania komunikacji IP.)

## ARP Spoofing Prevention
- Implement dynamic ARP inspection. (Polish: **Zapobieganie oszustwom ARP**: Wdroż dynamiczną inspekcję ARP.)

## DNS Security (DNSSEC, DoH, DoT)
- Use DNSSEC to ensure data integrity and authenticity in DNS responses. (Polish: **Bezpieczeństwo DNS** (DNSSEC, DoH, DoT): Używaj DNSSEC, aby zapewnić integralność danych i autentyczność odpowiedzi DNS.)

## DHCP Security
- Secure DHCP communications using DHCP snooping. (Polish: **Bezpieczeństwo DHCP**: Zabezpiecz komunikację DHCP za pomocą podsłuchiwania DHCP.)

## TCP/UDP Attack Prevention
- Implement rate limiting and proper input validation. (Polish: **Zapobieganie atakom TCP/UDP**: Wdróż ograniczenie prędkości i odpowiednią walidację danych wejściowych.)

## HTTPS/TLS Best Practices
- Use strong ciphers and enforce HTTPS everywhere. (Polish: **Najlepsze praktyki HTTPS/TLS**: Używaj silnych szyfrów i wymuszaj HTTPS wszędzie.)

## FTP Security
- Avoid using FTP; prefer SFTP or FTPS. (Polish: **Bezpieczeństwo FTP**: Unikaj używania FTP; preferuj SFTP lub FTPS.)

## Email Security
- Use encryption for email communications. (Polish: **Bezpieczeństwo e-mail**: Używaj szyfrowania dla komunikacji e-mail.)

## Network Monitoring
- Continuously monitor network traffic for suspicious activities. (Polish: **Monitorowanie sieci**: Ciągle monitoruj ruch sieciowy w poszukiwaniu podejrzanych działań.)

## Firewall Configuration
- Properly configure firewalls to block unauthorized access. (Polish: **Konfiguracja zapory sieciowej**: Poprawnie skonfiguruj zapory, aby zablokować nieautoryzowany dostęp.)

## VPN Setup
- Use strong encryption protocols for VPNs. (Polish: **Ustawienie VPN**: Używaj silnych protokołów szyfrujących dla VPN.)

## Attack Scenarios and Defenses
- Regularly update defense mechanisms based on emerging threats. (Polish: **Scenariusze ataków i obrony**: Regularnie aktualizuj mechanizmy obronne w oparciu o pojawiające się zagrożenia.)

## Security Checklist
- Maintain a security checklist and review it regularly. (Polish: **Lista kontrolna bezpieczeństwa**: Utrzymuj listę kontrolną bezpieczeństwa i regularnie ją przeglądaj.)
